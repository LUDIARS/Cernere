LUDIARS Authentication Platform

Cernere

汎用認証プラットフォーム & データリレーサーバー。複数の認証方式（OAuth / パスワード / Passkey / MFA）、組織・チーム管理、外部サービスの動的登録とユーザーデータ委託、そして WebSocket ベースのリアルタイムメッセージリレーを提供する。

LUDIARS の全サービスは Cernere を通じてセッションを確立する。Cernere は 常時接続セッション（Always-Connected Session）を基盤とし、 破壊的操作は認証済みかつ接続中のセッションからのみ受け付ける。 公開 REST は認証境界（/auth・/api/auth/* 等）に限られ、 データの参照・変更は WebSocket セッション上の module_request で実行される。

役割 — このサービスは何をするか

🔑

統合認証 (IdP)

GitHub / Google OAuth、bcrypt パスワード、WebAuthn Passkey、TOTP/SMS/Email MFA。さらに OIDC Provider として外部 RP に ID トークンを発行する。

🛰️

常時接続セッション基盤

WebSocket 上で 30 秒 ping / 10 秒 pong による生存検証。切断は即 SessionExpired に遷移し、以降の破壊的操作を遮断する。

🧩

サービス動的登録

外部サービス（Schedula / Nuntius 等）を managed_projects に登録し、各サービスのスキーマから project_data_<key> テーブルを実行時生成する。

🗂️

個人データ単一情報源

OAuth トークンやユーザーデータを Cernere に集約。サービスはユーザーの代わりに Cernere へ委託保管し、モジュール単位のオプトアウトを尊重する。

📡

メッセージリレー

同一ユーザーのセッション間リレー（クロスデバイス同期）と、service adapter 同士の peer 直結（relay_pairs + challenge 検証）。

📝

監査ログ

全 module_request（成功・失敗とも）を operation_logs に記録し、完全な監査証跡を提供する。

設計の柱 — セキュリティモデル

Cernere のセキュリティは 3 本の柱に基づく。トークンは単体では無効で、生存中の接続と組み合わさって初めて操作が許可される。

① 常時接続検証

持続接続と ping/pong により、トークン窃取・セッションハイジャックを継続的に無効化する。

② 強固な認証

接続確立時に JWT / セッション ID を必須化。未認証アップグレードは即拒否。

③ 破壊的操作の遮断

認証済みアクティブセッション以外からの削除・上書き・権限変更を完全にブロックする。

破壊的操作の 4 層防御

レイヤー検証内容失敗時
Layer 1Cookie / Bearer Token 検証401
Layer 2Redis セッション TTL チェック（7 日）401
Layer 3ユーザー状態検証（LoggedIn 必須）403
Layer 4リソース所有権・ロールチェック403

認可モデル

システムレベル

admin初回ログインユーザー。プロジェクト定義・登録の管理。
general一般ユーザー。

組織レベル

owner組織作成者。組織削除・ownership 移譲。
admin / maintainerメンバー管理・プロジェクト有効化。
member読み取り専用。自己退出のみ。
